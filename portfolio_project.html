Project Overview

This project investigates a fundamental question in quantitative finance: Can mean-variance optimization produce repeatable out-of-sample alpha? Using a diversified basket of 8 ETFs, I implemented a classic Sharpe ratio maximization framework and tested whether optimized portfolios maintain their edge when deployed on unseen data.

The answer: mostly no. While the optimizer produced impressive in-sample results (3.04% outperformance vs S&P 500), out-of-sample performance degraded significantly due to parameter instability and regime changes. The project became a valuable lesson in overfitting and the importance of realistic expectations for mean-variance approaches.

Methodology

Asset Universe

8 diversified ETFs across asset classes:

  • SPY: S&P 500 (US Large Cap Equities)
  • IEF: 7-10 Year Treasury Bonds
  • LQD: Investment Grade Corporate Bonds
  • TLT: 20+ Year Treasury Bonds
  • GLD: Gold
  • VNQ: US Real Estate (REITs)
  • VEA: Developed Markets ex-US
  • VWO: Emerging Markets

Optimization Framework

Classic mean-variance approach with constraints:

  • Objective: Maximize Sharpe ratio
  • Constraints: Long-only, weights sum to 1, max 35% per asset
  • Training period: 2018-01-01 to 2022-01-01
  • Test period: 2022-01-01 to 2024-01-01
  • Annualization: 252 trading days

Implementation Variants

Tested two approaches:

  • RB (Rebalanced): Portfolio rebalanced daily to target weights
  • BH (Buy-and-Hold): Weights drift naturally after initial allocation

Note: No transaction costs included - this makes results optimistic.

Diagnostic Analysis: Why Did It Fail?

1. Mean Reversion

Assets that performed best in-sample showed mean reversion out-of-sample:

  • IEF (7-10Y Treasuries): Strong train performance ‚Üí weak test performance
  • LQD (IG Credit): Received 35% weight but underperformed in test period
  • VWO (EM): Allocated 0% weight, but outperformed in test period

2. Correlation Shifts

The correlation structure changed materially between train and test periods. The optimizer exploited correlations that didn't persist:

  • Stock-bond correlation shifted from negative to positive (2022 rate regime)
  • Diversification benefits assumed in optimization didn't materialize
  • Assets weighted heavily for "diversification" moved in tandem during test

3. Regime Dependence

2018-2021 (train) was a low-rate, QE-supported environment. 2022-2023 (test) saw aggressive Fed tightening. The parameters estimated in one regime proved unstable in another.

Key Learning: The optimizer found patterns in the training data that were artifacts of that specific period rather than durable relationships. This is classic overfitting.

Technical Implementation

Optimization Setup

Used SciPy's minimize function with:

  • Sequential Least Squares Programming (SLSQP) method
  • Negative Sharpe as objective (minimize negative = maximize positive)
  • Linear constraints for weights (0 ‚â§ w ‚â§ 0.35, sum(w) = 1)
  • Initial guess: equal weights

Portfolio Return Calculation

Two methods implemented:

  • Rebalanced (RB): Daily returns = dot(weights, asset_returns)
  • Buy-and-Hold (BH): Compound asset values with drifting weights

Metrics Calculated

  • Annualized return and volatility
  • Sharpe ratio (assuming 0% risk-free rate for simplicity)
  • Maximum drawdown
  • Cumulative returns
  • Per-asset contribution to portfolio return

Key Takeaways

What Worked

  • Clean implementation: Code structure made it easy to diagnose issues
  • Comprehensive diagnostics: Return attribution and correlation analysis revealed the problems
  • Train/test split: Honest evaluation prevented overly optimistic conclusions
  • Multiple variants: Testing RB vs BH showed implementation choices matter

What Didn't Work

  • Point estimates: Using single-period mean/covariance proved unstable
  • No regime awareness: Optimizer couldn't adapt to structural market changes
  • Static weights: Set-and-forget allocation failed when relationships shifted
  • Overfitting: Maximizing Sharpe on noise rather than signal

Lessons for Future Work

  • Walk-forward testing essential for realistic expectations
  • Consider shrinkage estimators or Bayesian approaches for parameter estimation
  • Regime detection could improve robustness
  • Transaction costs would make results even worse - must be included
  • Simple equal-weight often beats optimization out-of-sample
Most Important Learning: This project was primarily a lesson in overfitting. The optimizer exploited noise in the training data rather than finding stable signals. Mean-variance optimization requires careful parameter estimation, regime awareness, and realistic expectations - it's not a magic bullet.

Future Improvements

  • Rolling window approach: Re-optimize monthly with expanding or rolling windows
  • Robust estimators: Use shrinkage for covariance (Ledoit-Wolf) and means
  • Risk parity: Compare against equal risk contribution approach
  • Transaction costs: Model realistic costs and turnover constraints
  • Factor models: Use factor exposures rather than raw returns
  • Regime indicators: Switch between allocations based on macro regime
